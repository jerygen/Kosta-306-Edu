# Antigravity Vibe Coding 지침서

본 지침서는 Antigravity(AI)가 해당 게시판 프로젝트를 Vibe Coding 방식으로 개발할 때 준수해야 할 개발 규칙과 가이드를 정의합니다.

## 1. 개발 환경 및 초기화
- 프론트엔드는 **React + Vite** 환경으로 구성합니다.
- 프로젝트 초기화는 비대화형(non-interactive) 방식으로 진행해야 합니다. 
  - 예: `npx -y create-vite@latest . --template react`
- 필수 의존성 패키지 설치: `npm install axios react-router-dom`
- 실행 스크립트: `npm run dev`

## 2. 코드 스타일 및 프로젝트 구조
- **함수형 컴포넌트(Functional Components)** 및 **React Hooks** 사용을 강제합니다.
- 컴포넌트를 논리적인 단위로 분할하여 재사용성을 높입니다.
- 추천하는 `src/` 폴더 구조:
  - `src/components/`: 버튼, 모달, 입력창 등 재사용 가능한 UI 요소
  - `src/pages/`: 라우팅되는 메인 뷰 (BoardList, BoardDetail, BoardForm)
  - `src/api/`: Axios 인스턴스 및 서버 API 통신 로직 분리

## 3. API 통신 가이드 (Axios)
- 백엔드와의 통신에는 `axios`를 사용하며, 모든 요청과 응답 데이터 포맷은 **JSON**입니다.
- API 호출 코드는 컴포넌트 내부에 하드코딩하지 않고, 별도의 서비스 파일(`src/api/board.js` 등)로 분리합니다.
- 비동기 처리(`async/await`)를 사용하며, 통신 실패 시 `try-catch`로 에러를 잡고 사용자에게 친절한 에러 메시지를 표시해야 합니다.
- 백엔드가 완성되지 않았다면, Mock 데이터를 사용하거나 더미 서버를 구성해 프론트엔드를 먼저 완성할 수 있도록 유연하게 대처합니다.

## 4. 디자인 및 UI/UX 원칙 (매우 중요)
- 프로덕션 레벨의 모던하고 세련된 UI를 제작해야 합니다.
- 기본 브라우저 스타일은 지양하며, 현대적인 타이포그래피, 적절한 여백, 호버(Hover) 애니메이션, 그림자 효과 등을 추가해 시각적 만족도를 높이세요.
- 상태(로딩 중, 데이터 없음, 에러 발생)에 따른 UI 피드백을 반드시 구현합니다.
- TailwindCSS 사용을 요청받지 않았다면, 바닐라 CSS(`index.css`, `App.css` 또는 CSS Modules)를 활용하여 정교하게 스타일링합니다.

## 5. Vibe Coding 진행 단계 (프롬프트 가이드)
개발을 진행할 때 다음의 단계별 흐름을 따라 코딩을 지시하는 것이 효율적입니다.

1. **셋업 (Setup):** "현재 폴더에 React Vite 프로젝트를 생성하고 기본 라이브러리를 설치해."
2. **API 및 라우팅 설정:** "API 엔드포인트를 호출하는 axios 함수들을 만들고, react-router-dom으로 페이지 라우팅 틀을 잡아줘."
3. **목록 구현:** "목록 화면을 구현해줘. 로딩 상태와 목록 데이터 출력 디자인을 모던하게 적용해."
4. **상세 및 등록/수정 구현:** "상세 페이지와 글쓰기 폼을 만들어줘. 유효성 검사와 저장 버튼 클릭 시 API 호출이 완료되면 목록으로 이동하게 해줘."
5. **디자인 및 최적화 폴리싱:** "전체적인 CSS 디자인을 더 세련되게 다듬고 디테일을 추가해줘."
