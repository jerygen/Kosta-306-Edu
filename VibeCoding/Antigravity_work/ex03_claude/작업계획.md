# [작업 목표]

퍼블리싱된 정적 웹 리소스(HTML, CSS, JS)를 React(Vite) 기반의 프로젝트로 변환합니다. 메인 페이지(`index.html`)를 분석하여 재사용 가능한 컴포넌트로 분리하고, 요구사항에 맞춰 styled-components를 활용한 스타일링으로 전환합니다.

## ⚠️ 사용자 검토 필요 (User Review Required)

- **styled-components 적용 방식**: 기존 정적 리소스에 있는 CSS(`reset.css`, `common.css`, `content.css`) 내용들을 모두 styled-components 코드로 변환할지, 아니면 글로벌 스타일과 개별 컴포넌트 스타일로 어떻게 나눌지 확인이 필요합니다. (초기 계획은 기존 스타일을 분석해 styled-components의 `GlobalStyle` 및 개별 컴포넌트 스타일로 완전히 분리하는 것입니다)
- **Swiper 라이브러리**: 기존에 사용된 jQuery 기반의 `swiper4.2.min.js` 대신, React 환경에 맞는 `swiper/react` 모듈을 설치하여 변경할 예정입니다.

## ❓ 오픈 퀘스천 (Open Questions)

- 라우터 추가 시 `react-router-dom` 라이브러리를 사용할까요? (요구사항에 '라우터 경로 : /'가 있어 설치 예정입니다)
- 이미지나 폰트 같은 에셋(asset)들은 `public` 폴더로 이동할지, 아니면 `src/assets`에서 관리할지 선호하시는 방식이 있나요? (기본적으로 `src/assets` 또는 `public/img`로 이동할 계획입니다)

## 🛠 제안하는 변경 사항 (Proposed Changes)

### 1. 프로젝트 초기화 및 라이브러리 설치
- Vite를 사용하여 React 프로젝트(JavaScript) 생성
- 필요 라이브러리 설치: `styled-components`, `react-router-dom`, `swiper`

### 2. 폴더 구조 설계
```text
src/
 ┣ components/       # 공통 및 UI 컴포넌트
 ┃ ┣ common/
 ┃ ┃ ┣ Header.jsx    # 상단 GNB 및 로고 영역
 ┃ ┃ ┗ Footer.jsx    # 하단 정보 영역
 ┃ ┗ main/           # 메인 페이지 전용 컴포넌트
 ┃   ┣ VisualSlider.jsx     # 메인 비주얼 슬라이더 (bg1~bg5)
 ┃   ┣ ProductSection.jsx   # 제품 소개 (round_wrap)
 ┃   ┣ ConstructionCase.jsx # 시공 사례 (color section)
 ┃   ┗ BoardSection.jsx     # 공지사항, 고객센터, 견적문의
 ┣ pages/
 ┃ ┗ MainPage.jsx    # '/' 경로에 맵핑될 메인 페이지
 ┣ styles/           # 글로벌 스타일 (styled-components)
 ┃ ┣ GlobalStyle.jsx # reset.css, common.css 기반
 ┃ ┗ theme.js        # 공통 색상, 폰트 사이즈 등
 ┣ App.jsx           # 라우팅 설정 (react-router-dom)
 ┗ main.jsx
```
